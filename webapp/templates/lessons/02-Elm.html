Execute code in the browser

HTML is not a programming language in the sense that we can not compute anything. We can only represent static data.

The traditional (and currently almost the only) way to execute something in the browser is to write Javascript. However, this language, prototyped in 10 days, is hard to use efficiently:

  • errors messages can be as useless as "undefined is not a function",
  • type coercion is pretty complex and often leads to hard-to-debug situations,
  • there are a lot of syntactic constructions, some of them being almost identical (but not totally!). Moreover, we can not use directly the newest syntaxes since the old browsers won't support it.

To compensate thoses weaknesses, developpers have produced a lot of libraries and frameworks. Nowdays, a standard project in Javascript includes a stack of several of those libraries and frameworks. E.g. React+Redux+Axios+Typescript... Each of those library is great, but each adds complexity to the development and do not give strong garantees about the correct execution of the application.

Still, the browsers only executes Javascript (and now Webassembly, but it is another story). The solution I propose in this course is to write our applications in Elm, a totally different language, which compiles to Javascript. That means we will write the code in Elm, run a compiler which will transform the code in Javascript, and finally execute the code produced by the compiler.
Don't be scarry, the compilation step will be integrated in the tools we will use!
