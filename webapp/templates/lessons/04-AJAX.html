Json and Elm decoders

It is time to make the server and the client to communicate by another way of HTML. We are still doing HTTP, so the schema is always the same: the client send a request to the server and the server send back a response.
However, in a Single Page Application (SPA) the page is not reloaded and those requests are performed via Javascript (in our case, this Javascript is compiled from Elm).
The most currently used format to perform this communication is JSON (JavaScript Object Notation). Other formats can also be used:

  • XML
  • Protobuf
  • GraphQL
